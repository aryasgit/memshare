threat model

Memshare assumes the relay is hostile. Since you run the relay yourself, the threat reduces to "your own laptop". Browsers exchange ECDH keys and relay ciphertext through Node; nothing is written to disk, nothing leaves the LAN.

The Loop

01
Download the zip for your OS and double-click the file inside.
+
A small installer script clones the repo, runs npm install once, and starts the relay in local mode. Your browser opens http://localhost:8787 automatically.
02
Hit start a new room. Hand the URL to a teammate.
+
Memshare prints a LAN URL like http://192.168.1.5:8787. Anyone on the same Wi-Fi can open it. Verify the four-quartet fingerprint with your teammate out-of-band so a malicious LAN device can't sit in the middle.
03
Paste code, drop files, type messages.
+
Fenced markdown blocks (```python) are syntax-highlighted. Files are encrypted in 256 KB chunks before they leave your browser. Each peer derives its own pair key via ECDH — the relay only ever sees opaque blobs.
04
Close the tab. It's gone.
+
Room state is held in memory only and is cleared when the last peer disconnects. File blobs expire after one hour even if you forgot. Nothing is written to disk.

Two ways in. Double-click installer,
or clone the repo.

Download the zip, unzip, double-click memshare-mac.command.

The first time, macOS may say "cannot be opened because it is from an unidentified developer". Right-click → Open → confirm, or go to System Settings → Privacy & Security and click "Open Anyway".

The installer needs Node 20+ and git. If either is missing it'll tell you exactly what to install.

Download the zip, unzip, double-click memshare-windows.bat.

The first time, SmartScreen may warn "Windows protected your PC". Click "More info" → "Run anyway".

The installer needs Node 20+ and Git for Windows. If either is missing it'll tell you exactly what to install.

memshare-windows.zip↓

Vanilla Node + vanilla browser JS. No build step.

git clone https://github.com/aryasgit/memshare.git
cd memshare
npm install
npm run local

The crypto module is public/js/crypto.js. The relay is server/relay.js. Both readable in twenty minutes. scripts/smoke.mjs spawns two virtual peers and round-trips an encrypted message + a 200 KB file.

No accounts.
No telemetry.
No database.

The whole relay is <400 lines of JavaScript. Room state lives in memory and dies with the process. File blobs are wiped after an hour. We can't read your messages because we never have your keys — and you control the only relay that touches them.
