threat model

Memshare assumes the server is hostile. It exchanges ECDH public keys, relays ciphertext, and stores nothing in plaintext. If we get subpoenaed, we have ciphertext to hand over.

The Loop

Open a room.
Hand off the code. Start typing.

01
Open the app and tap start a new room.
+
A six-character code appears. Your browser generates a fresh ECDH P-256 keypair just for this session; the private half never leaves the tab.
02
Hand the code to your teammate over any channel.
+
Slack, signal, a sticky note. The code by itself is enough to join the room — verifying you're talking to the right person is done out-of-band with the four-quartet fingerprint shown on screen.
03
Paste code, drop files, type messages.
+
Fenced markdown blocks (```python) are syntax-highlighted. Files are encrypted in 256 KB chunks before they leave your browser. Everything streams to peers in real time.
04
Close the tab. It's gone.
+
The server's room state is held in memory only and is cleared when the last peer disconnects. File blobs expire after one hour even if you forgot. Nothing is written to disk.

Install

One process. Either on the public host,
or on your laptop.

Join the hosted Memshare. No install on either side.

  1. Open memshare.app in two tabs (or send the URL to a teammate).
  2. One side taps start a new room; share the 6-character code.
  3. Verify the fingerprint quartet over Signal / a phone call / in person.

Run Memshare on your machine. Teammates on the same LAN connect over your IP. The server you run is the only one involved.

git clone https://github.com/you/memshare.git
cd memshare
npm install
npm run local

Memshare prints a LAN URL like http://192.168.1.5:8787. Hand that to a teammate on the same network.

It's vanilla Node + vanilla browser JS. No build step.

npm run dev   # node --watch
# server/  → Fastify + ws relay
# public/  → HTML/CSS/ES modules
# scripts/smoke.mjs → encrypted round-trip test

The crypto module is public/js/crypto.js. The relay is server/relay.js. Read both in twenty minutes.

No accounts.
No telemetry.
No database.

The whole server is <400 lines of JavaScript. Room state lives in memory and dies with the process. File blobs are wiped after an hour. We can't read your messages because we never have your keys.
